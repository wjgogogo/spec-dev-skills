# 预设风格详细定义

当用户没有明确指定视觉风格时，从下面几种预设中选一种；如果用户给了参考图，优先向参考图靠拢，而不是机械套预设。

## `semantic-tech`

- 适用：普通技术流程图、架构图、模块关系图
- 特征：低饱和语义配色，保留"不同语义不同颜色"的可读性，但整体更克制、更偏文档图
- 不要使用高饱和主色；优先使用灰化后的雾紫、灰蓝、灰绿、陶土橙、粉棕红
- 建议配色：
  - 紫色：边框 `#9a86a8`，背景 `#f6f1f8`
  - 蓝色：边框 `#7fa4b4`，背景 `#eef5f7`
  - 绿色：边框 `#7f9f93`，背景 `#eef4f1`
  - 橙色：边框 `#b7937e`，背景 `#f8f2ee`
  - 红色：边框 `#c19a96`，背景 `#f8efee`
  - 靛蓝/起止节点：边框 `#8c88b4`，背景 `#f3f1f8`
- 文字仍统一使用 `#374151`
- 箭头建议比节点更克制；可以继续跟随语义色，但应再降一档饱和度
- 建议：2-3 种强调色即可，不要把每个节点都染成不同颜色；复杂图优先统一箭头颜色，只保留节点语义色

## `claude-docs`

- 适用：希望接近 Claude 官方文档插图、产品文档配图、轻量说明图
- 整体气质：暖白画布、灰米色起止节点、浅绿主流程节点、蓝灰主箭头、浅绿回环线、浅橙提示框
- 画布背景：`#f8f5f0`
- 起止节点：边框 `#d4d0cb`，背景 `#ece9e4`
- 主流程节点：边框 `#c8d8c7`，背景 `#dce8d9`
- 虚线容器：边框 `#ded9d3`，背景透明
- 主流程箭头：`#767b86`
- 回环箭头：`#9dcba3`
- 提示框：边框 `#e6a178`，背景 `#faf0e8`
- 提示文字/橙色强调：`#8f4b27`
- 标题与正文：`#292524`
- 线条建议：`roughness: 0-1`，主箭头用实线，提示/中断路径用虚线
- 布局建议：优先横向主流程 + 虚线框住 loop/agentic loop + 底部或侧边放提示框

## `minimal-paper`

- 适用：更克制的设计说明图、论文配图、内部技术文档
- 特征：暖灰或白底，1-2 种中性色，弱化语义色块
- 建议配色：
  - 画布：`#ffffff` 或 `#f8f5f0`
  - 主节点边框：`#3f3f46`
  - 主节点背景：`#fafaf9` 或 `#f5f5f4`
  - 次要说明：边框 `#a8a29e`，背景 `#ffffff`
  - 正文：`#374151`

## `warm-poster`

- 适用：概念图、模型图、轻海报感的说明图
- 特征：暖色大底、低饱和高对比标题、节点数量少、文字短
- 建议：只保留 1 个主色背景和 1 个浅色文字色，避免复杂箭头和过多边框

## 选择规则

1. 用户提到 "Claude 文档风格 / Anthropic 风格 / 参考 Claude 官网图" 时，优先使用 `claude-docs`
2. 用户只说"技术图、流程图、架构图"，默认使用 `semantic-tech`
3. 用户强调"克制、文档感、论文感"时，优先使用 `minimal-paper`
4. 用户强调"概念海报、视觉风格强"时，优先使用 `warm-poster`
5. 一旦用户提供参考图，预设只作为起点，最终以参考图的配色、布局、线条粗细、留白关系为准
